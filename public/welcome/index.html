Research into LLM privacy capabilities

DeepResearch.se is an innovation-and-research project into the privacy capabilities of LLM applications — how far a real, useful research assistant can be pushed toward provable privacy, and where that trades against capability. The site itself is the proof, and the goal is to take that proof as far as possible: a fully open-sourced, independently verifiable pair — DeepResearch.Se/cure, where the research runs entirely in your browser and the server is absent from every data path, and DeepResearch.Se/rver, the signed-in tier where a server buys capability and protects what it handles with encryption and policy. It's still experimental and nowhere near production-ready; the code is open (MIT), so every privacy claim is yours to verify — and to reuse. It began as a weekend build through the Claude Code iPhone app, kept in the build story.

First time here? What this does — and doesn't

It does

  • Plan its own web searches, run them, and answer with numbered citations — within a time budget you set.
  • Read the documents and photos you attach, keep work in projects, quiz you on a topic, look up places on the map, and even explain its own source code.
  • Run code for you in an experimental Linux sandbox that boots right inside your browser tab.
  • Do all of that entirely in your browser, on your own OpenAI, Groq or Berget key — no account, and no cost to anyone, that in-browser sandbox included.

It doesn't

  • Sell anything — no ads, no trackers. This is an open exploration of AI-assisted architecture and the build process itself, not a commercial product.
  • Put the server in the loop when you run it in your browser — DeepResearch.Se/cure never sees your chats or your keys, by construction.
  • Take open signups. Hosted extras — EU-hosted models, web search, cloud sync at DeepResearch.Se/rver — run on cost-bearing APIs, not-for-profit, so access is approval-gated: sign in with the account button (top right) and await the operator's OK.
