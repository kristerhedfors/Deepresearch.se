Notifications

Everything needing admin attention in one place: sign-ins awaiting approval and operational issues (Berget errors, wallet balance, dropped streams) surfaced automatically as they happen. Each item explains the issue and what to do about it. Repeated occurrences of the same operational issue bump a counter instead of piling up duplicate rows; dismiss once handled and a recurrence brings it back.

Users

Accounts are created automatically on first Google sign-in — the address configured as ADMIN_EMAIL becomes admin, everyone else a regular user with the default quota. Manage roles, status, and per-user quotas here.

Security risks

The open items from SECURITY-RISKS.md (the living register in the repo). Items are collapsed to their headers — tap a header to open its full details (summary, votes, score, note), drag the ⠿ grip to reorder. In the fix-order view the top-to-bottom order you drag into IS the explicit priority: the FIXED order the Claude Code security-fix loop works through on its next round (unprioritized items follow by votes, then documented severity). You can still set a priority number by hand inside an item. Switch the view between that fix order and documented severity.

Features

The feature backlog from FEATURES.md (the living roadmap in the repo) — the second channel that feeds a Claude Code loop (the first is Security). Items are collapsed to their headers — tap a header to open its full details (summary, votes, effort, note), drag the ⠿ grip to reorder. In the build-order view the top-to-bottom order you drag into IS the explicit priority: the FIXED order the feature-build loop works through on its next round (unprioritized items follow by votes, then documented impact). Shipped/dropped items sink to the bottom as the record. Switch the view between that build order and documented impact.
