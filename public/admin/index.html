Security risks

The open items from SECURITY-RISKS.md (the living register in the repo). Items are collapsed to their headers — tap a header to open its full details (summary, votes, score, note), drag the ⠿ grip to reorder. The top-to-bottom order you drag into IS the explicit priority: the FIXED order the Claude Code security-fix loop works through on its next round — top is first in line to be fixed. Reset order clears every explicit priority, returning the list to its default: documented severity (votes break ties first, if any were cast).

Reset order clears every explicit priority, returning the list to its default: documented impact (votes break ties first, if any were cast). Shipped/dropped items sink to the bottom as the record.

Web search service

Which service actually runs the research pipeline's web searches. The default is Exa (the built-in). Point it at your OWN self-hosted service — a SearXNG instance or any Exa-compatible endpoint — to keep search queries off a third party's retention. Only the short AI-derived query ever leaves the server; never the conversation. See the local-web-search skill for ready-to-run service recipes (SearXNG, a Playwright crawler, and more).
