Se/cure — deep research secure — keeps everything in this browser and calls the model provider directly. The hosted tier with live web search, documents and cloud storage is DeepResearch.Se/rver.

Account · this browser

Se/cure has no accounts and no server-side data. Your chats, projects and API keys live only in this browser. The one thing that ever leaves your device is the model call itself — sent straight to your chosen provider (OpenAI, Groq or Berget) on your own key. This site's server is never in the path, so it cannot read or log any of it.

Configuration — your API key and the execution sandbox — lives under Settings (the gear in the header).

Settings · this browser
Provider API keys

Your key goes from this browser straight to the provider — never through this site's server. With a saved project it's stored only inside your sealed blob, in this browser.

Execution sandbox (bash) Experimental

Boots a real Linux in this browser so the assistant can run shell commands to answer you (e.g. “run ls”). Everything runs on your device. The first use downloads a Linux image, so it's slow to start.

Developer mode

Unlocks introspection mode: ask about this site's own implementation (“how are you built?”, “show me src/pipeline.js”) and the assistant answers from a snapshot of the exact source code this deployment runs — fetched as a public static file, so the server still sees none of your conversation. With the sandbox also on, the whole source tree is mounted at /src in the Linux VM for real shell exploration.
